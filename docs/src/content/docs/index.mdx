## Why marky?

<CardGrid stagger>
  <Card title="Mermaid-first" icon="pencil">
    Validates `lang=mermaid` code blocks via `@mermaid-js/parser`. Diagram
    syntax errors surface inline, before they reach your docs site.
  </Card>
  <Card title="Built on unified/remark" icon="add-document">
    Uses the mdast AST pipeline — position-aware, composable, and compatible
    with 300+ existing remark plugins.
  </Card>
  <Card title="markdownlint compatible" icon="setting">
    Drop-in bridge for `.markdownlintrc` configs. Run `crackdown migrate` to
    generate a `crackdown.config.ts` from your existing setup in seconds.
  </Card>
  <Card title="Auto-fix built in" icon="open-book">
    `crackdown lint --fix` applies safe mechanical fixers (trailing spaces, hard
    tabs). `--dry-run` shows what would change without touching files.
  </Card>
</CardGrid>
